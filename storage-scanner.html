Architecture

How Storage Intelligence Platform works

Agents run directly on each storage node and publish scan results to the central control plane over a standard HTTPS connection. The control plane never pulls data from agents — agents always initiate — which means network segmentation and firewall rules remain simple and agents continue scanning even when the server is temporarily unreachable.

Deployment

Built for complex storage environments

Three properties that matter in enterprise storage infrastructure.

Incremental by design

The first scan of a node establishes a baseline. Every subsequent scan only processes files that have changed — new, modified, or deleted. Large NAS, Lustre, and ZFS filesystems that would take hours to full-scan complete incrementally in minutes, with no measurable I/O impact on production workloads.

Agents survive server outages

Scanner agents never block on server availability. If the control plane is unreachable, scan jobs complete locally and results are held until the connection resumes. No data is lost, no scan windows are missed, and no manual intervention is needed after a maintenance window or network partition.

Air-gap and offline ready

Docker images for both the server and agent are provided as loadable tarballs for environments with no outbound internet access. The agent auto-update mechanism can serve packages from a local server path, eliminating the need for any external package repository during operation.

Distribution

Available packages

Every release ships the same code in five formats. Choose the one that fits your deployment tooling.

🆕 DEB — Debian / Ubuntu

Native packages for Ubuntu 22.04, Ubuntu 24.04, and Debian 12. Both the server and agent are available as .deb files — install with dpkg or include in an Ansible playbook.

📦 RPM — RHEL / Rocky / Alma

RPM packages for RHEL 8/9, Rocky Linux 9, and AlmaLinux 9. Install directly with rpm -i or integrate into Satellite / Ansible automation.

💾 Docker — Server and Agent

Docker image tarballs for air-gapped environments. Load with docker load. The server image includes PostgreSQL client tools. Compose file and .env example included in each release.

🪟 Windows ZIP — Agent

Windows Server 2019+ agent distribution. Extract and run as a service for mixed Windows/Linux storage estates.

📦 Full Bundle

Every release also ships as a single audit-platform-{version}.tar.gz containing all packages, install scripts, documentation, and a CycloneDX 1.4 SBOM. One archive — everything needed for an air-gapped rollout.

✅ Release Certification

Every release passes a 7-gate certification model before publication: build verification, package integrity, unit tests, migration check, licence gate, SBOM generation, and checksum signing. The evidence bundle ships inside the full archive.
