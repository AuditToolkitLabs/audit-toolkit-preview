Regulated Environments

Built for environments where compliance is non-negotiable and data cannot leave.

The deployment model that regulated environments require

Cloud-hosted security tools can create a compliance problem where audit findings, asset inventory, vulnerability data, and remediation records include sensitive operational information. This suite deploys entirely within your infrastructure with no normal-operation data transmission to external services. Telemetry is disabled by default and, when enabled, transmits only non-sensitive operational metadata.

Air-gap and offline deployment

  • ✓Offline MSI installer for restricted Windows environments
  • ✓Agent deployment without outbound internet access
  • ✓Advisory and vulnerability data can be supplied via local customer-managed files
  • ✓Deployment supported on isolated network segments

Evidence your auditors will accept

  • ✓SHA-256 checksums on every artifact
  • ✓MANIFEST.json and SHA256SUMS.txt in every evidence package
  • ✓Framework mapping to NIST SP 800-53, CIS, PCI-DSS references
  • ✓Auditor-ready ZIP packages for Big 4 audit firm delivery
  • ✓Digital signatures for chain-of-custody documentation
  • ✓30/60/90 day compliance trajectory with before/after metrics
  • ✓Full remediation audit trail with rollback history

Source code protection for long-term deployments

The BSL-licensed products in this suite include automatic MIT conversion: Audit Admin Toolkit and Asset Command Center on 14 February 2030, Linux Security Lite on 9 February 2031. After those dates, deployed instances and source code carry no licence restrictions.

Relevant frameworks and standards

  • ✓NIST SP 800-53 Rev 5 control families
  • ✓DISA STIG coverage across Linux, Windows, and network platforms
  • ✓CIS Benchmark Level 1 and Level 2 alignment
  • ✓PCI-DSS finding-to-control mapping
  • ✓SOC 2 audit trail and access control evidence
  • ✓ISO 27001 Annex A reference mapping
