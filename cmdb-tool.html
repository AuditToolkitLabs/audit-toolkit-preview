Ingestion Layer

CMDB API Data Collection Tool

The CMDB API Data Collection Tool is the ingestion and normalisation layer of the AuditToolkit. It collects infrastructure data via 40+ integrations across cloud, network, identity, and enterprise systems, delivering normalised records to the core platform or standalone CMDB.

Capabilities

Key Features

📡

CMDB API Data Collection Tool

Multi-platform infrastructure data ingestion and normalisation

  • ✓40+ integrations across cloud, network, identity, and enterprise systems
  • ✓Agents for Windows (.NET 8), Linux (Go), macOS, BSD, and containers
  • ✓Dual-mode operation: standalone CMDB or integrated feeder to the core platform
  • ✓Canonical payload contract (schema_version 2026.05.1) for normalised asset and vulnerability records
  • ✓Dead-letter queue with retry and backoff for reliable delivery
  • ✓SSO claim and attribute mapping for enterprise identity integration
  • ✓Full tenant and workspace governance model

Data Flow

How CMDB Data Collection works

The data collector ingests structured data from external APIs and agent-based collection, normalising results into a canonical schema for the central platform or standalone CMDB operation.
