CMDB Asset Platform

Next-generation asset discovery, inventory visibility, and vulnerability context aligned to CMDB workflows. A unified inventory foundation for cloud and hypervisor environments — with enhanced auto-discovery, risk-linked context, and a normalised operational source of truth for security and operations teams.

In development. The CMDB Asset Platform is in active development. The capability areas and outcomes described here reflect the current development direction. Features may change before general availability.

Platform Outcomes

One inventory baseline for audit and control execution

The CMDB Asset Platform is designed to resolve the fragmentation problem across cloud, hypervisor, and on-premises inventory — producing a consistent, enriched operational baseline teams can act on.

🏗

Consolidated Inventory

Bring disparate cloud and virtualisation inventories into one normalised model. Eliminate duplication and gaps across AWS, Azure, GCP, VMware, Proxmox, Nutanix, and KVM.

🛡

Risk-Linked Context

Connect asset inventory directly to vulnerability and CVE data. Surface risk prioritisation and remediation context alongside each asset record — not in a separate tool.

✅

Operational Confidence

Provide audit and operations teams with a single inventory baseline for control execution, audit evidence, and governance reporting.

Capability Areas

Discovery, normalisation, and enrichment at every layer

Built to cover the full stack — from cloud management APIs through to on-premises hypervisors — with a unified API layer for downstream integration.

☁

Multi-Cloud Discovery

Automated discovery workflows for AWS, Azure, and GCP. Credential-based scope configuration with per-account and per-subscription inventory pull, normalised into a common asset model.

🖥

Hypervisor Coverage

Inventory pathways for VMware vCenter, KVM/libvirt, Proxmox, and Nutanix. Guest-level visibility including network, storage, and compute attributes per virtual machine.

📡

Inventory API Layer

Unified REST APIs for inventory readout, integration with external CMDB platforms (ServiceNow, Jira Assets), and downstream automation workflows.

🔍

Vulnerability Context

CVE correlation and CVSS v3 scoring per asset record. Risk prioritisation feeds integrated directly into the inventory view — aligned with the CMDB Data Collection Tool's correlation engine.

📄

CMDB-Aligned Normalisation

Data normalisation and enrichment designed around standard CMDB field models. Consistent classification, naming, and relationship mapping across all discovery sources.

📈

Audit & Governance Reporting

Exportable inventory snapshots, coverage reports, and enrichment audit trails. Designed for compliance evidence and governance review workflows.

Product Relationship

Built on the CMDB Data Collection Tool foundation

The CMDB Asset Platform is a next-generation evolution of the inventory and discovery capabilities in the CMDB API Data Collection Tool. While the Data Collection Tool is available now at GA, the Asset Platform extends those foundations with enhanced auto-discovery, a richer asset model, and deeper CMDB alignment.

  • ✓CMDB Data Collection Tool — available now at GA (v0.2.6). 29+ connectors, CVE correlation, OWASP Top 10 PASS.
  • ✓CMDB Asset Platform — in development. Enhanced auto-discovery, normalised CMDB model, multi-cloud inventory APIs.
  • ✓Shared vulnerability engine — both tools share the same CVE correlation and CVSS scoring approach.
