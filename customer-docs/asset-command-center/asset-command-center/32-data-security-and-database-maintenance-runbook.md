# 32. Data Security and Database Maintenance

## Sensitive data categories

- user and role records
- report-scope and team metadata
- target-system credentials stored through the product
- audit and operational logs
- local reporting and snapshot data
- license state and entitlement metadata

## Operational guidance

- protect PostgreSQL backups and configuration backups
- keep encryption and secret material outside source control
- restrict database administration to authorized staff
- review retention for logs and exported reports under customer policy
- follow normal PostgreSQL maintenance standards for the deployment
