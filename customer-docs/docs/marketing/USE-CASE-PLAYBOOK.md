# Use Case Playbook

## Use Case 1: Security Hardening Baseline At Scale

Audience:

- Security engineering
- Infrastructure operations

Outcome:

- Baseline audits across Linux and mixed estates with clear remediation pathing

Primary products:

- Audit-Tool
- AuditToolkit Linux Security Lite

CTA:
Run a baseline cycle and compare pass/fail trend movement over 30 days.

## Use Case 2: Asset And Configuration Visibility For Risk Reviews

Audience:

- IT operations
- Compliance and audit teams

Outcome:

- CMDB-grade inventory and metadata collection through API and agent patterns

Primary products:

- CMDB API Data Collection Tool
- Asset Command Centre

CTA:
Stand up inventory collection for top-tier systems and map high-risk gaps.

## Use Case 3: Switch Exposure Prioritization

Audience:

- Network security
- Network operations

Outcome:

- Correlate device identity, firmware, advisories, and CVE exposure for
  actionable remediation planning

Primary products:

- Switch Exposure Center

CTA:
Run advisory refresh and build a firmware remediation priority list by risk.

## Use Case 4: Remote Assurance For Distributed Operations

Audience:

- Platform operations
- Assurance teams

Outcome:

- Controlled remote action workflows with verification artifacts

Primary products:

- Audit Assurance Node

CTA:
Pilot remote assurance execution for one environment and validate bundle output.

## Use Case 5: Governance-Ready Documentation And Release Evidence

Audience:

- Product operations
- Release engineering

Outcome:

- Consistent release-documentation process with controlled sync and traceability

Primary products:

- AuditToolkit Docs (source authority)
- AuditToolkit Control Centre (internal governance context)

CTA:
Adopt central docs sync and validate docs evidence in every release bundle.
