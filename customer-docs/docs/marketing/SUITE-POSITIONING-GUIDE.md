# Suite Positioning Guide

## Core Positioning

AuditToolkit is a self-hosted security and operations platform suite for teams
that need audit visibility, asset intelligence, and practical remediation
workflows without surrendering control of infrastructure data.

## Suite Story

AuditToolkit combines:

- broad host audit coverage
- centralized reporting and control-plane workflows
- product-specific specialist tools for Linux, asset inventory, network switch
  exposure, and remote assurance
- package-first deployment for enterprise environments

## Value Themes

1. Operational control

- Self-hosted deployment model for regulated or data-sensitive environments.

1. Practical coverage

- Multi-domain collection and audit capabilities across server, endpoint,
  inventory, and network contexts.

1. Fast time-to-value

- Native package support, clear runbooks, and release-bundle workflows.

1. Confidence and governance

- Security scoring, validation gates, and docs-release controls that improve
  auditability.

## Positioning Guardrail

Use direct competitive language only with proof points tied to specific
capabilities. Position as strong in self-hosted hardening/compliance and
integrated reporting, with transparent roadmap language where parity is still
in progress.

## Website Narrative Arc

- Problem: teams are spread across disconnected security, asset, and
  operations tools.
- Shift: centralize evidence and action paths while keeping deployment control.
- Outcome: faster risk visibility, clearer ownership, and repeatable response.
