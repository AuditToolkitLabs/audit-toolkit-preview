# Messaging Pillars

## Pillar 1: Keep Control Of Your Security Data

Claim:
AuditToolkit is designed for self-hosted operation, so teams can keep
infrastructure and audit evidence under their own control boundaries.

Supporting points:

- Native package and on-prem deployment patterns.
- Local evidence and operational runbook workflows.

## Pillar 2: Connect Audit, Inventory, And Exposure Workflows

Claim:
AuditToolkit unifies host audits, asset intelligence, and network exposure into
an actionable platform story.

Supporting points:

- Suite products cover core host, CMDB/API, Linux-first, switch exposure, and
  remote assurance domains.
- Central documentation and bundle sync improve consistency.

## Pillar 3: Move From Findings To Action Faster

Claim:
Teams can move from findings to prioritized remediation with clear ownership and
repeatable execution paths.

Supporting points:

- Remediation script ecosystems and control mappings.
- Operational runbooks, validation scripts, and release gates.

## Pillar 4: Build Trust Through Verifiable Process

Claim:
AuditToolkit emphasizes measurable validation, policy boundaries, and
transparent constraints.

Supporting points:

- Security scorecards and readiness checks.
- Documentation source-of-truth and release-bundle policy controls.

## Pillar 5: Scale With Integration-Ready Interfaces

Claim:
API and export pathways support downstream SIEM, workflow, and reporting
integration.

Supporting points:

- Broad endpoint and API surfaces in core products.
- Queue-based export and contract-oriented integration design.
