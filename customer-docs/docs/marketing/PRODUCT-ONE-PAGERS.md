# Product One-Pagers

## Audit-Tool (Core Platform)

Positioning:

Self-hosted security audit, compliance, discovery, and reporting platform.

Best fit:

- Security teams needing broad host/control coverage with centralized evidence.
- Organizations that require package-driven deployment and local control.

Proof themes:

- Large audit and remediation script footprint.
- Extensive API surface for integration and reporting workflows.
- Security hardening posture and measurable validation metrics.

## Audit Assurance Node

Positioning:

Remote elevation and assurance execution framework for distributed runtime
operations.

Best fit:

- Teams needing controlled remote action orchestration.
- Environments requiring evidence bundles and repeatable runtime checks.

Proof themes:

- Environment-based configuration model.
- Verification and bundle-integrity workflow support.

## AuditToolkit Linux Security Lite

Positioning:

Linux-first standalone audit toolkit for host hardening and compliance checks.

Best fit:

- Linux-centric estates with local-first operations.
- Teams needing lightweight deployment and SIEM/export pathways.

Proof themes:

- Linux-only scope clarity.
- Control-plane contract and queue export model.
- Tier-gated enterprise and integration extensions.

## CMDB API Data Collection Tool

Positioning:

API-based CMDB collection and reporting platform with managed-agent support.

Best fit:

- Teams building inventory, metadata, and endpoint collection pipelines.
- Organizations that need API-centric data ingestion and reporting workflows.

Proof themes:

- API and collector model separation.
- Package and targeted-update deployment options.

## Switch Exposure Center

Positioning:

Switch-focused inventory and exposure reporting platform for network
infrastructure.

Best fit:

- Network and security teams needing firmware/advisory/CVE visibility for
  switch estates.
- Environments requiring vendor-specific advisory enrichment pipelines.

Proof themes:

- Vendor connector strategy with API-first posture.
- Exposure correlation model tied to remediation readiness.

## Asset Command Centre

Positioning:

Legacy-focused asset collection, normalization, and reporting service with
Linux primary deployment.

Best fit:

- Teams consolidating cross-platform inventory and operational telemetry.
- Organizations needing migration-safe packaging and compatibility continuity.

Proof themes:

- Primary Linux service path with broader legacy collection remit.
- Customer/internal documentation boundary clarity.

## AuditToolkit Control Centre (Internal)

Positioning:

Internal-only operations interface for platform health, workspace management,
and release coordination.

External marketing rule:

Do not present as customer product surface. Mention only as internal governance
capability where required for trust/process narratives.
