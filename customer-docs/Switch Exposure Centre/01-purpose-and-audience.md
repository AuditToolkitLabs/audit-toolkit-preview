# Purpose and Audience

## What this documentation is for

Switch Exposure Center is a switch-focused CMDB and exposure reporting
platform. This documentation explains how the product is used, how it is
operated, and where the boundaries are for customer deployment and support.

## Intended readers

- Network and infrastructure operators
- Security and vulnerability management teams
- Platform administrators
- Support and service desk staff
- Technical approvers and procurement reviewers

## Scope

The product focuses on network switches and adjacent equipment. It brings
together inventory, firmware state, advisories, CVEs, and remediation status
so teams can work from one operational view.

## What the product covers

- Multi-vendor switch inventory
- Advisory and vulnerability correlation
- Remediation-oriented reporting
- API-first collection with fallback transports where needed
- A web console for operator review and administration

## What the product does not try to be

- A generic IT asset platform for all infrastructure types
- A ticketing system
- A replacement for security tooling that performs active exploitation
- A full network monitoring platform

## Documentation conventions

- Vendor names and product labels are written in their common form.
- Configuration examples describe the product's current behavior rather than
  a hypothetical future state.
- Security and operational guidance is written for the current codebase and
  release posture.
