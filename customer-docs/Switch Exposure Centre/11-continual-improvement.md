# Continual Improvement

## Improvement goals

The product should continue to improve in the areas that affect customer value
most directly: data quality, coverage, operational simplicity, and clarity of
reports.

## Common improvement themes

- Better source coverage for vendor advisories
- More robust collection fallbacks
- Clearer remediation prioritisation
- Better export and reporting workflows
- Reduced manual administration effort

## Feedback loop

Teams should capture:

- Missing data fields
- Source or connector failures
- Reporting questions from users
- Operations pain points
- Requests for new vendor support

## Release learning

Each release should record what changed, what improved, and what remains open so
operators can see whether the product is moving in the right direction.
