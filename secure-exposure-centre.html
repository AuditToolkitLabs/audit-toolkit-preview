Secure Exposure Centre

Secure Exposure Centre provides switch-centered exposure visibility for multi-vendor network estates. It collects inventory and firmware posture, correlates advisories and CVEs, and delivers remediation-oriented reporting for network operations and security teams.

Naming note. Release artefacts and customer docs are currently published under the product path Switch Exposure Centre while this website surface uses the commercial name Secure Exposure Centre.

What It Delivers

Exposure intelligence for network switch estates

This tool is built for teams that need vulnerability and advisory visibility aligned to actual switch firmware and platform inventory, not generic endpoint scanning outputs.

Multi-vendor collection

  • ✓Live API-backed collection for Cisco, Juniper, Arista, Aruba, and Dell.
  • ✓Initial SAN scaffolding for Brocade and Cisco MDS.
  • ✓API-first model with transport fallback planning for SSH, NETCONF, and SNMP.

Exposure and remediation context

  • ✓Vendor advisory normalization and CVE/CVSS correlation.
  • ✓CISA KEV awareness where feeds are reachable.
  • ✓Remediation-oriented reporting tied to firmware and platform identity.

Operationally safe deployment shape

  • ✓Health and readiness endpoints for service checks.
  • ✓Single-host Ubuntu Server LTS production baseline.
  • ✓Package workflows and manifest verification for release integrity.

Governance-ready workflows

  • ✓Documented operations, security, maintenance, and support runbooks.
  • ✓Advisory refresh automation with external scheduler model.
  • ✓Customer-facing legal, assurance, and EULA documentation.

Supported Vendors

Current MVP coverage

Live API-backed collection

  • ✓Cisco IOS-XE and NX-OS (PSIRT RSS live feed enabled)
  • ✓Juniper JunOS
  • ✓Arista EOS
  • ✓Aruba AOS-CX
  • ✓Dell OS10

SAN connector support

  • ✓Brocade (connector scaffolding present)
  • ✓Cisco MDS (connector scaffolding present)
Advisory source model: Advisory data sources include Cisco PSIRT RSS (live), CISA KEV (live), and NVD (live). Juniper, Arista, Aruba, Dell, Brocade, and Cisco MDS advisory access requires customer-supplied vendor feed credentials or a customer-managed local export file. Vendor-gated advisory access is a customer responsibility and the platform provides the ingestion infrastructure.
Advisory automation: Advisory refresh runs on an external scheduler model (Windows Task Scheduler, cron, or container orchestrator). Per-vendor refresh can be enabled independently and refresh API responses report source status with fallback state per vendor.
